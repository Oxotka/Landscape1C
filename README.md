# Ландшафт технологий 1С

Интерактивная карта экосистемы технологий для разработчика на 1С и около 1С — наследник линейного каталога [StackTechnologies1C](https://github.com/Oxotka/StackTechnologies1C).

Цель — дать не просто список, а **навигацию** по двум осям: функциональной роли (разработчик / администратор / аналитик / тестировщик; администратор включает devops/эксплуатацию) и контексту работы (франчайзи / инхаус / продукты / проекты), плюс фильтры по зрелости (`базовое` / `продвинутое` / `нишевое`), лицензии, происхождению (важно для импортозамещения) и доступности на рынке РФ, с карточками технологий, связями между ними и отдельной страницей графа.

## Статус

🚧 Прототип. Реализуется **свой статический фронтенд** (папка `app/`). Подробности и история решений — в **[docs/TZ.md](docs/TZ.md)**.

## Подход

Сначала пробовали готовый движок [landscape2](https://github.com/cncf/landscape2) (как у [CNCF Landscape](https://landscape.cncf.io)), но он не поддерживает две кастомные оси фильтрации (роль + контекст) и русскоязычный UI — поэтому от него отказались. Делаем свой статический фронтенд: HTML/CSS/JS без сборки, данные в `app/data.js`.

## Структура репозитория

- `app/` — **фронтенд (живой сайт):**
  - `index.html` + `app.js` — главная: фильтры, доска карточек, поиск, сортировка
  - `data.js` — единственный источник данных (`window.LANDSCAPE`: `categories`, `blocks`, `axes`, `items`)
  - `detail.js` — общая детальная модалка (используется и главной, и графом)
  - `graph.html` + `graph.js` — страница «Граф связей» (analogs/depends)
  - `council.html`, `methodology.html` — экспертный совет и методология; фото в `people/`
  - `editor.html` + `editor.js` — визуальный редактор разметки (см. ниже)
  - `styles.css`, `logos/` — оформление и логотипы (SVG)
- `scripts/` — `validate.js` (проверка целостности данных), `build.js` (сборка `dist/`), `serve.py` (пишущий сервер для редактора)
- `docs/TZ.md`, `docs/METHODOLOGY.md` — техзадание/история решений и методология разметки осей
- `.github/workflows/deploy.yml` — публикация `app/` на GitHub Pages

## Локальный предпросмотр

Сборка не нужна — это статика. Запусти любой статический сервер из папки `app/`:

```bash
cd app && python3 -m http.server 8123
# открой http://127.0.0.1:8123/
```

## Редактор и проверки

Визуальный редактор разметки `app/editor.html` пишет правки прямо в `app/data.js` — для записи нужен пишущий сервер (обычный `http.server` так не умеет):

```bash
python3 scripts/serve.py     # http://127.0.0.1:8123/editor.html
node scripts/validate.js     # проверка целостности данных (оси, блоки, логотипы, связи)
node scripts/build.js        # сборка dist/ для ручной выгрузки на хостинг
```

## Публикация на GitHub Pages

Подготовлено, но **не активно** (репозиторий приватный). Чтобы включить:

1. Сделать репозиторий публичным.
2. Settings → Pages → Source: **GitHub Actions**.
3. Запустить workflow «Deploy to GitHub Pages» вручную, либо раскомментировать триггер `push` в `.github/workflows/deploy.yml` для автодеплоя.

Адрес проектного сайта: `https://oxotka.github.io/Landscape1C/` (пути в `app/` относительные — работает без доп. настроек). Свой домен подключается позже через `CNAME` без переделок.
